# PR / Submission Checklist

Use this checklist before submitting the project or creating a PR for review.

- [ ] README.md is complete and accurate
- [ ] .env.example present in backend
- [ ] .gitignore configured
- [ ] Backend starts with `node server.js`
- [ ] Frontend starts with `npm start`
- [ ] Demo credentials included in README
- [ ] Screenshots & demo links added to README (if deployed)
- [ ] No console errors in browser
- [ ] Basic manual tests passed (create, update, delete, search, notes, status)
- [ ] All changes committed and pushed to GitHub
- [ ] PR description includes deployment instructions
